# 规则验证、影子运行与信号寿命

任何底部启动、加速、涨停、连板、轮动阶段或市场状态规则，在进入正式候选前必须具有固定目标、截点和版本。

## 状态

| 状态 | 允许输出 |
|---|---|
| `experimental` | 描述性信号、观察名单、触发与失效条件 |
| `shadow` | 记录虚拟候选和实际可成交性，不向用户声称已验证优势 |
| `validated` | 仅在时间分离样本外验证、成本和成交约束通过后用于正式研究层级 |
| `suspended` | 历史优势衰减、数据漂移或规则失效，停止产生正式候选 |

没有验证记录时默认 `experimental`，不能自动视为 `validated`。

## 发布前要求

- 分开 `T+1触及涨停`、`T+1收盘为正`、`T+5超额为正`；
- 固定预测截点、特征、阈值、复权方式和 `ruleset_version`；
- 按时间切分开发、验证和样本外区间，不随机打乱；
- 保留历史ST、退市、停牌和板块成分；
- 计入手续费、滑点、涨停买不到和跌停卖不出；
- 报告样本量、基准率、收益分布、MFE/MAE和置信区间；
- 按市场状态、板块阶段、证据等级和流动性分层。

运行 `scripts/research_tracker.py` 记录信号和结果。报告必须按 `objective + ruleset_version + horizon` 分组，禁止把不同规则版本合并后宣称表现提升。

```powershell
python scripts/research_tracker.py --db research.sqlite3 report --min-sample 30
```

`minimum_reached` 只表示达到描述性样本门槛，不代表样本外验证已经完成。

## 信号寿命

跟踪信号产生后在T+1、T+3、T+5等时点是否仍有优势，以及延迟一天执行后优势是否消失。至少按以下状态记录：

- `active`：触发条件仍成立；
- `aging`：优势衰减或拥挤上升；
- `invalidated`：价格、板块、证据或市场条件失效；
- `expired`：超过规则预先定义的有效期；
- `suspended`：该规则在近期样本中失效。

不得在看到结果后修改原始信号、入选原因、失效条件或版本。单日失败只进入复盘，规则调整必须基于时间分离样本。
