# 数据质量门槛

在计算任何股票研究分之前先执行本规范。数据质量不合格时停止评分；不要用更多指标掩盖数据问题。

## 1. 数据质量分

| 维度 | 权重 | 核查内容 |
|---|---:|---|
| 时效性 | 25 | 业务时间、披露时间、抓取时间、接口更新频率 |
| 一致性 | 25 | 字段公式、跨接口、跨来源及财务勾稽 |
| 完整性 | 20 | 必填字段、历史覆盖、缺失原因 |
| 一手来源 | 15 | 交易所公告、定期报告、公司/政府原文 |
| 可追溯性 | 10 | 来源、请求路径、版本、响应哈希 |
| Schema 稳定性 | 5 | 字段类型、单位、可空性及接口变更 |

处理规则：

- `90–100`：允许正常研究评分。
- `80–89`：允许分析，最终研究分乘以 `0.9` 并标记中等置信度。
- `60–79`：只输出事实和缺口，不进入优先候选。
- `<60` 或存在硬错误：停止股票评分。

数据质量分与股票机会分必须分开显示。

## 2. 时间和 Point-in-Time

每条记录保存：

| 字段 | 含义 |
|---|---|
| `source_time` | 行情对应的成交/快照时间 |
| `period_end` | 财务报告期末 |
| `disclosure_time` | 信息首次公开时间 |
| `revision_time` | 更正或修订公开时间 |
| `fetched_at` | 本系统抓取时间 |
| `available_at` | 该记录最早可用于模型的时间 |

回测使用 `available_at <= prediction_cutoff` 的版本。保留旧版本，不用当前修订值覆盖历史。记录历史 ST、停复牌、更名、上市/退市、股本及复权因子变化，防止幸存者偏差和前视泄漏。

交易日、集合竞价、午间休市、盘后、停牌和节假日使用交易日历判定。盘中实时快照超过供应商更新周期两倍时标记 `stale`；盘后不得用“距现在超过五分钟”误判收盘数据。

## 3. 缺失值语义

禁止把空值自动转为零。使用以下状态：

- `missing_source`：供应商未返回；
- `not_disclosed`：公司未披露；
- `not_applicable`：指标不适用；
- `permission_denied`：套餐无权限；
- `stale`：数据超出允许时效；
- `api_error`：请求失败；
- `actual_zero`：确认数值为零。

接口返回空数组时，先核查代码、日期、交易日、权限和更新时点，再判断业务上是否确实无数据。

## 4. 行情校验

对单股实时行情至少检查：

```text
最低价 <= 开盘价、最新价 <= 最高价
涨跌额 ≈ 最新价 - 昨收价
涨跌幅 `pc` ≈ (最新价 / 昨收价 - 1) × 100%
振幅 `zf` ≈ (最高价 - 最低价) / 昨收价 × 100%
成交量、成交额、换手率、市值不得为负
总市值 ≈ 最新价 × 总股本
流通市值 ≈ 最新价 × 流通股本
```

初始容差：价格一个最小报价单位、涨跌幅 `0.05` 个百分点、市值 `1%`。若字段保留位数较低，允许在报告中解释并调整容差，不能静默忽略。

同一抓取窗口内优先交叉比较：单股实时、20股批量、全市场实时和最新日线。价格差超过一个报价单位、涨跌幅差超过 `0.05` 个百分点或成交量差超过 `1%` 时标记冲突。先比较 `source_time`，不要对不同时间的快照直接判错。

分钟线汇总日线时明确是否包含集合竞价。开高低收必须一致，成交量/额初始容差 `0.5%`；超限时停止使用该日聚合特征。

## 5. 财务校验

关键指标必须使用“智兔 + 独立来源”验证，最终以交易所或公司法定披露裁决。两个使用同一上游数据的聚合平台不算独立来源。

```text
误差率 = |来源A - 来源B| / max(|来源A|, 极小值)
```

- `<=1%`：一致；
- `1%–5%`：警告，列出两个数值和可能口径差异；
- `>5%`：硬停止，回查原始财报和附注。

核查口径：单季度/累计、合并/母公司、归母/净利润、扣非/非扣非、元/万元/亿元、报告期/披露期、原始披露/更正。

可计算时执行：

```text
资产 ≈ 负债 + 所有者权益
期末现金 ≈ 期初现金 + 现金净增加额
毛利率 ≈ (营业收入 - 营业成本) / 营业收入
资产负债率 ≈ 总负债 / 总资产
市值 ≈ 实际股价 × 对应时点股本
```

不得用当前总股本验证历史市值，也不得用当前复权因子重写历史原始成交价。

## 6. 复权和公司行为

- 当前快照使用不复权实际成交价。
- 历史走势和区间价格比较统一使用前复权。
- 总回报使用包含分红的后复权或自行按公司行为重建。
- 每条历史序列记录 `adjustment` 和 `factor_version`。
- 分红、送转、配股、增发、回购和股本变更按除权日/登记日版本化。

同一计算序列不得混用复权方式。分钟级接口只支持不复权时，不要伪造分钟复权结果。

## 7. API 错误和重试

| 状态 | 含义 | 处理 |
|---|---|---|
| `401` | 当日请求量超过版本限制 | 不重试；暂停非关键任务并报警 |
| `402` | Token 无效 | 不重试；提示重新配置 |
| `404` | 路径/资源错误 | 校验代码、日期、接口版本 |
| `429` | 频率超限 | 指数退避并加入随机抖动 |
| 网络超时/5xx | 临时故障 | 最多重试 2–3 次 |

连续失败触发熔断。可回退缓存时必须显示缓存的 `source_time` 和 `fetched_at`，不能伪装成实时数据。日志记录接口名、HTTP 状态、耗时、响应哈希和错误类型，但绝不记录 Token 或完整带 Token URL。

## 8. 数据血缘

标准化记录至少包含：

```text
provider
endpoint
instrument
source_time / period_end / disclosure_time
fetched_at
adjustment
schema_version
response_hash
quality_score
quality_flags
```

保留原始响应的不可变副本，标准化表和衍生指标可重建。供应商升级后用固定样本做 Schema 漂移测试；发现新增、删除、改名、类型或单位变化时暂停相关特征。

## 9. 输出要求

每份分析先输出：

1. 数据截止时间和预测截点；
2. 数据质量分与等级；
3. 使用来源和复权方式；
4. 跨来源差异；
5. 缺失、过期、权限和接口错误；
6. 是否允许进入股票评分。

任何硬错误都必须先于选股结论显示。
