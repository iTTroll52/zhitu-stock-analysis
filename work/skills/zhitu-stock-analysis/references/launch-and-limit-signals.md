# 启动、加速与涨停结构

使用 `scripts/short_term_signals.py` 识别四类研究信号：`bottom_start`、`accelerating`、`limit_up`、`consecutive_limit_up`。信号描述当前结构，不表示未来涨停概率。

## 1. 输入要求

每个候选至少提供：

```json
{
  "code": "600000",
  "name": "示例",
  "sector": "行业",
  "sector_stage": "strengthening",
  "market_regime": "repairing",
  "quality_score": 90,
  "evidence_score": 65,
  "business_support": "confirmed",
  "comparable_volume_ratio": 1.6,
  "quote": {
    "p": 10.5,
    "yc": 10.0,
    "o": 10.1,
    "h": 10.6,
    "l": 10.0,
    "pc": 5.0,
    "hs": 6.0,
    "lb": 1.6,
    "cje": 500000000
  },
  "bars": [
    {"t": "2026-07-15", "o": 10.1, "h": 10.6, "l": 10.0, "c": 10.5, "v": 100000}
  ],
  "pool": {
    "is_limit_up": false,
    "limit_up_price": 11.0,
    "lbc": 0,
    "fbt": null,
    "lbt": null,
    "zbc": 0,
    "zj": null
  }
}
```

`bars` 至少包含预测截点前 20 根完整日线，统一使用前复权进行趋势比较。实际成交价和精确涨停价使用不复权价格。盘中成交放大优先传入历日同一时刻的 `comparable_volume_ratio`；不能拿上午成交量与历史全天成交量比较。

涨停池记录已经确认涨停时，可将池中的实际涨停成交价 `p` 标准化为 `limit_up_price`。非涨停池候选不得擅自按10%粗算，必须处理证券类别、无涨跌幅限制情形和价格取整规则。

## 2. 底部启动

底部不是历史最低价。`bottom_start` 同时要求：

- 位于近20日区间下部；
- 5日均线开始上拐且价格重新站上；
- 当日上涨受控，不是已经涨停或连续加速；
- 同时点成交量温和放大；
- 板块处于启动、修复或强化；
- 数据质量通过。

缺少业绩、订单、产能或公司催化证据时，可以保留技术观察，但不能进入优先级候选。

## 3. 即将加速

`accelerating` 要求：

- 价格和5/10/20日均线呈多头结构；
- 最近多日动量为正，但当日涨幅和成交放大仍处于受控区间；
- 接近近20日强势区，而不是远离趋势后的极端脉冲；
- 板块处于强化或加速，龙头与中军不是单票独舞；
- 有公司催化或业绩逻辑支持。

成交量突然极端放大、连续大幅上涨、高开低走或板块宽度收缩时，标记“高潮/兑现风险”，不能因为加速分高而追涨。

## 4. 涨停与连板

`limit_up` 必须同时有当日最高价和按照证券类别、前收盘价及交易所取整规则得到的精确涨停价。不得只用“涨幅约10%”替代。

`consecutive_limit_up` 还要求当期连板数至少为2，并检查：

- 首次和最后封板时间；
- 开板次数；
- 封单资金相对成交额；
- 换手率和是否一字板；
- 板块涨停宽度及梯队；
- 连板高度和可成交性。

一字板、四板以上、高炸板次数、极端换手或公司证据不足时，只能进入观察名单，并明确“不可假设可以买到”。

## 5. 输出和复盘

运行：

```powershell
python scripts/short_term_signals.py candidates.json --top 5 --output signals.json
```

四类信号分别排序，同一股票按 `连板 > 涨停 > 加速 > 底部启动` 选择主标签。输出必须包含信号分、条件真假、数据来源、禁止追涨标记和失效条件。

分别统计每类信号的 T+1 正收益、T+1 触及涨停、T+5 超额收益和最大有利/不利波动。没有足够样本外记录前，不得把分数换算成概率。
