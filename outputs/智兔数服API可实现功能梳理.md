# 智兔数服 API：可实现功能梳理

> 整理日期：2026-07-11  
> 依据：智兔数服的 [沪深 A 股](https://www.zhituapi.com/hsstockapi.html)、[沪深指数](https://www.zhituapi.com/hsindexapi.html)、[北交所 A 股](https://www.zhituapi.com/bjdataapi.html) API 文档。  
> 本文只梳理数据可支持的产品功能，不代表接口已逐项实测，也不构成投资建议。

## 1. 结论：适合做什么

这套 API 足以支持一个以 **“研究、筛选、复盘、风险提示”** 为中心的 A 股分析网站，覆盖沪深 A 股、主要沪深指数及北交所 A 股。最值得优先做的不是“预测涨停”或“自动荐股”，而是：

1. **股票研究页**：行情、估值、财务、股东、分红、管理层集中展示。
2. **选股/筛选器**：按市值、估值、涨跌幅、技术指标、财务指标和风险标签进行筛选。
3. **市场与板块看板**：指数、概念、行业、涨跌股池、涨停池和市场广度。
4. **自选股与预警中心**：关注股票的分钟级行情、财报/分红/股东变化和条件预警。
5. **基本面体检与投资复盘工具**：把数据转成需要进一步核查的“红旗”和“待验证假设”。

接口覆盖行情与基本面，但文档中未见新闻正文、公告全文、研报全文、交易账户流水、真实盘口逐笔委托队列或下单交易接口。因此，第一版不应承诺新闻解读、券商级盘口、自动化实盘交易或完整的个人收益归因。

## 2. 数据能力地图

| 数据域 | 沪深 A 股 | 沪深指数 | 北交所 A 股 | 可支持的产品能力 |
|---|---|---|---|---|
| 标的主数据 | 股票列表、新股、ST、概念/一级市场板块及成分 | 主要指数列表 | 股票列表 | 搜索、分类、标的库、板块归属 |
| 实时行情 | 单股、全市场、最多 20 只多选、分时/逐笔交易 | 指数实时交易 | 单股实时交易 | 行情页、自选股、盘中看板、条件预警 |
| 历史行情 | 多周期 K 线、历史分时、行情指标 | 多周期 K 线、历史分时 | 多周期 K 线 | K 线、收益/回撤、相对强弱、回测数据底座 |
| 技术指标 | MA、MACD、KDJ 等 | MACD、KDJ 等 | 文档未列技术指标 | 技术形态看板、规则筛选、信号提示 |
| 公司基本面 | 公司资料、管理层、分红、利润/现金流、股东、财务三表及财务指标 | 不适用 | 财务三表 | 公司研究页、质量筛选、财务红旗 |
| 交易与情绪 | 涨停/跌停/炸板等股池、资金/成交类数据 | 指数成交数据 | 基础行情 | 短线情绪面板、市场热度、风险提示 |

## 3. 可以实现的功能清单

### A. 市场首页与行情中心

**适合做成网站首页。**

| 功能 | 展示/计算内容 | 主要数据依据 | 更新建议 |
|---|---|---|---|
| 全市场概览 | 上涨/下跌、涨停/跌停、成交额、换手率、热点榜 | 沪深全市场实时数据、涨跌股池、涨停池 | 盘中 1–10 分钟 |
| 指数行情卡 | 上证、深证、创业板及其他主要指数的价、涨跌、成交 | 指数列表、指数实时交易 | 盘中刷新 |
| 市场宽度 | 上涨家数占比、涨停/跌停比、不同市值区间表现 | 全市场实时行情 + 股票列表 | 盘中刷新 |
| 涨跌榜/异动榜 | 涨跌幅、涨速、量比、换手率、成交额、60 日/YTD 涨跌幅排名 | 沪深实时交易字段 | 盘中刷新 |
| 新股日历 | 申购日、发行价、申购上限、配市值、中签率、上市日 | 新股日历 | 每日收盘后 |
| ST 风险名单 | 风险警示股票及自选股命中提示 | 风险警示股票列表 | 每日收盘后 |

**价值**：让用户先看“市场发生了什么”，再决定是否研究个股；不要把榜单直接包装成买入建议。

### B. 个股研究页（最优先）

**一只股票一个完整档案。**

1. **基础资料区**：代码、名称、交易所、所属概念/板块、公司概况。
2. **实时行情区**：现价、涨跌幅、开高低、成交量/额、换手、量比、振幅、涨速、动态 PE、PB、总/流通市值、60 日与年初至今涨跌幅。
3. **K 线与技术区**：5/15/30/60 分钟、日/周/月/年 K 线；不复权、前复权、后复权；MA、MACD、KDJ 叠加展示。
4. **经营质量区**：利润、现金流、资产负债表和财务主要指标的季度/年度趋势。
5. **股东与治理区**：十大股东、股东户数变化、高管任职历史；用于提示持股集中度和治理变化，不能据此断言资金意图。
6. **分红区**：近年来分红、除权除息与股权登记日，计算历史股息率时须说明价格口径。
7. **风险提示区**：ST、负利润/异常高 PE、经营现金流弱于利润、负债率抬升、商誉占比高、股东户数异常变化等“需要研究”的提示。

### C. 股票筛选器与观察清单

**适合做成用户每天打开的核心工具。**

可按以下条件组合筛选：

- 交易维度：交易所、板块/概念、总市值、流通市值、价格、成交额、换手率、量比、振幅、涨跌幅、60 日涨跌幅、年初至今涨跌幅。
- 估值维度：动态 PE、PB、历史区间位置（需自行由历史数据计算）。
- 财务维度：营收/利润增速、毛利率、净利率、ROE/ROA、资产负债率、经营现金流、每股收益、每股净资产等。
- 技术维度：均线多空排列、MACD 金叉/死叉、KDJ 区间、价格相对均线位置、突破近期高低点。
- 风险维度：是否 ST、是否新股、负 PE、负经营现金流、股东户数变化、分红缺失等。

建议筛选结果只显示 **“满足规则”**，同时展示每个条件的实际数值、数据时间与风险标签；不使用“必涨”“金股”等措辞。

### D. 板块、概念与主题热度

文档提供概念指数列表、一级市场板块列表和板块明细，可实现：

- 概念/行业目录与成分股页。
- 板块内个股涨跌、成交额、涨停数、平均换手率排行。
- “今日强势板块”和“持续走强板块”：以板块内标的实时行情和历史行情自行聚合计算。
- 个股关联主题：在个股页显示其所属板块与板块内相对表现。
- 主题轮动观察：按日/周/月比较板块涨跌、成交与波动。

注意：文档的板块来源标注为“券商数据”，正式上线前应确认其定义、成分调整频率、再分发与展示授权。

### E. 涨停生态与短线情绪工具

沪深文档含涨停股池以及相关涨跌股池数据，可做：

- 涨停池：首封/末封时间、连板数、封板资金、炸板次数、成交额、换手率、流通市值。
- 连板梯队：按连板数分层显示，形成每日情绪面板。
- 炸板复盘：按炸板次数、封板时间和换手率排序，作为波动风险观察。
- 涨停历史回放：文档注明涨停股池可按交易日查询，历史从 2019-11-28 起。
- 次日表现统计：将历史涨停池与后续日线自行拼接，计算开盘/收盘表现和回撤；必须标注样本、时间范围与幸存者偏差。

这部分适合“盘后复盘”定位。盘中数据并不等于可成交价格，也不能证明资金持续性。

### F. 指数与基准分析

指数 API 可实现：

- 指数搜索与行情：主要沪深指数的代码、名称、交易所、实时行情。
- 多周期指数图表：分钟、日、周、月、年线及成交量/额。
- 指数技术状态：MACD、KDJ 等指标。
- 相对收益：个股或自选组合相对指定指数的区间超额收益（由双方历史收盘价自行计算）。
- 市场环境提示：指数趋势、波动、成交变化；应使用“状态描述”，不应输出确定性择时指令。

### G. 财务体检、质量筛选与研究辅助

沪深 A 股有财务三表和财务主要指标；北交所文档列出财务三表。因此可实现：

- 三表趋势图：营业收入、归母/净利润、经营现金流、资本开支、总资产、总负债、货币资金等。
- 核心比率看板：EPS、每股经营现金流、ROE、ROA、销售净利率、资产负债率等。
- 同比/环比计算：以连续报告期自行计算增长率与趋势。
- 财务红旗：例如利润增长但经营现金流持续恶化、应收/存货增长显著快于收入、负债加速、商誉偏高、ROE 下滑。
- 质量筛选：定义“连续盈利 + 现金流覆盖 + 负债可控 + 估值处于阈值内”等透明规则。
- 公司研究卡：用户填写投资逻辑、估值假设、关键风险和证伪条件；接口数据自动补充客观事实。

**重要限制**：财务数据按报告期披露，不能把报告期末数据当成实时数据；页面应同时显示“截止日期”和“披露日期”。

### H. 股东、分红与公司治理观察

- 十大股东与持股结构的报告期对比。
- 股东户数变化趋势与人均持股变化（若所需分母可取得/计算）。
- 高管任职历史与职务变动时间线。
- 历年分红与除权除息日历。
- “分红稳定性”标签：需以多年记录、利润和现金流共同判断，不能只按单年股息率下结论。

### I. 自选股、提醒与个人复盘

接口可提供市场数据；用户的持仓、交易和研究结论需由网站自行存储。

- 自选分组：价值观察、财报季、短线复盘、北交所观察等。
- 条件提醒：价格/涨跌幅/成交额/换手率/估值/技术指标/财务更新/分红日期/ST 状态变更。
- 每日简报：自选股的涨跌、异常成交、估值变化、板块相对表现。
- 研究日志：买入前假设、关键证据、风险、预期持有期、证伪条件。
- 模拟组合：手工录入买卖，计算成本、持仓收益、最大回撤、相对指数表现；不涉及券商账户或自动下单。

## 4. 建议的产品版本

| 版本 | 目标 | 建议功能 | 暂不做 |
|---|---|---|---|
| V1：研究与自选 | 验证用户是否愿意持续使用 | 个股研究页、指数首页、自选股、日/周 K 线、财务趋势、分红/股东、基础筛选 | 预测、社区、自动交易 |
| V2：筛选与复盘 | 提升研究效率 | 组合筛选、板块页、财务红旗、相对收益、涨停盘后复盘、提醒 | 实时全市场高频扫描的复杂策略 |
| V3：个人决策系统 | 降低情绪交易 | 研究卡、模拟组合、交易日志、归因、仓位/集中度风险提示 | 代客理财、个性化买卖指令 |
| V4：专业数据产品 | 面向高级用户 | 数据导出、批量筛选、策略研究、历史统计 | 未确认授权的数据再分发 |

## 6. 数据更新与工程设计要点

1. **不要让前端直接携带 token 调接口**：token 应只保存在后端环境变量；由服务端做鉴权、缓存和限流。
2. **分层缓存**：实时行情按文档频率刷新；股票列表、ST 列表、财务、股东、分红按日/报告期入库；历史 K 线增量同步。
3. **明确数据时间**：每个卡片显示“数据截至/更新时间”，财务同时显示报告期与披露日。
4. **使用批量接口**：自选股实时展示可优先使用最多 20 只的多选实时接口；全市场实时接口文档标注仅限包年版和至尊版、且每分钟限 1 次。
5. **遵守频率与套餐限制**：文档多处列出不同套餐的每分钟请求上限；上线前应根据预计用户数核算缓存命中率与套餐。
6. **数据口径校验**：PE 为动态估算口径，不能与静态/TTM PE 混用；复权类型、交易所代码格式、单位（股/手/元）需要统一。
7. **留存原始数据与版本**：对财报、股东和成分股建立抓取时间与来源版本，便于回溯页面结果。

## 7. 合规与产品边界

- 网站应清楚标注数据来源、更新时间、延迟情况和风险提示。
- 避免收益承诺、确定性涨跌预测、针对特定用户的买卖指令、代客下单或保本宣传。
- 对“涨停”“资金流”“技术信号”使用观察/筛选/复盘表述，并提供反例与风险说明。
- 上线、收费、公开分发、二次展示或导出数据前，应核实智兔数服的服务协议、套餐权限及数据再分发授权；涉及投顾服务时应另行进行合规评估。

## 8. 文档中已确认的关键能力与限制

- 沪深股票列表、ST 列表、概念/一级市场板块等基础数据按日更新；新股日历含申购、发行、上市、中签等字段。
- 沪深涨停股池按交易日获取，文档标注盘中每 10 分钟更新；含连板、首末封、封板资金与炸板次数等字段。
- 沪深全市场实时交易为盘中每 1 分钟更新，但文档标注仅包年版/至尊版可用且每分钟限 1 次；多选实时接口单次不超过 20 只。
- 沪深个股的最新行情可取分钟至年线、多种复权方式；技术指标含 MA、MACD、KDJ。
- 沪深财务主要指标每日更新；公司三表、分红、十大股东和股东变化等适合做中长期研究。
- 指数 API 覆盖主要沪深指数列表、实时交易、历史行情与 MACD/KDJ。
- 北交所 API 覆盖股票列表、实时/历史行情与财务三表；页面目录未列出技术指标或公司治理/股东类模块，产品设计时需以实际接口响应再确认。

## 9. 下一步建议

先确定网站的首要用户：

1. **个人长期投资者**：优先做 V1 的个股体检、自选与财务趋势；这是最稳健的切入点。
2. **盘后复盘用户**：在 V1 基础上加入涨停生态、板块热度与历史统计。
3. **筛选效率需求强的用户**：再建设多条件选股器、提醒和数据导出。

在开始编码前，建议先向 API 方确认：套餐对应的接口权限、实时数据延迟、历史数据起点、是否允许缓存/公开展示/导出、板块成分更新机制，以及 QPS 超限的返回与计费规则。
